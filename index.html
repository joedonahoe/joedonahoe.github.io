2018.09.04 :: Get Git Set Up

I want each of these exercises to be done in one hour. I was planning on giving myself 30 minutes to get my Git environment setup and I'm 29 minutes in, and still haven't documented my Git commands or pushed back up to the server.

Here is how I got started:

  1. Open Git Bash. After installing Git, of course.
  2. Go to the local folder where you want to do your work. I did mine in simply c:\git
  3. Run git init. Then I did git config --global user.name "joedonahoe" and git remote add origin https://github.com/joedonahoe/joedonahoe.github.io. I had these in an earlier document, but I'm not exactly sure why I needed them.
  4. Now to pull the existing objects down. Run git clone https://github.com/joedonahoe/joedonahoe.github.io.
  5. And run git status. Nothing has changed. You can also run "dir -all" to see the files' current state.
  6. Make some changes. For me, I used "rm *" to throw away all of the old stuff. Then I created a new "index.html" in the folder.
  7. Run git status, and I have a lot of changes to send to the server.
  8. Then I ran git commit -a -m "2018 Reboot" and git push origin master.
